Working memory is what you're using right now to hold this sentence together while you read it. It's the brain's mental scratchpad — active, temporary, and remarkably small.

## Capacity: ~4 Chunks

George Miller's famous 1956 paper said "7 ± 2". More recent work (Cowan, 2001) revised this down to about **4 chunks**, maybe less.

A *chunk* is a unit of meaning. "FBI" is one chunk for an American, three unrelated letters for a child. Expertise is largely about building bigger, richer chunks.

> [!note] Chunking is the mechanism behind expertise
> A chess grandmaster doesn't see 32 pieces — they see 5-6 meaningful configurations. This is why experts can take in a position at a glance while novices are overwhelmed. See [[Deliberate Practice]] for how chunks are built.

## Cognitive Load

When working memory gets overloaded, learning stops. This is **cognitive overload**.

Three types (Sweller's Cognitive Load Theory):
- **Intrinsic** — the inherent complexity of the material itself
- **Extraneous** — complexity introduced by bad instruction/presentation
- **Germane** — productive effort that builds schemas

> [!tip] Practical implication
> If you're learning something hard, reduce every other source of cognitive load:
> - One topic at a time
> - No distractions
> - Clear, well-organized materials
> - Build on prior knowledge (prior knowledge = free working memory)

## Working Memory vs Long-term Memory

| | Working Memory | Long-term Memory |
|---|---|---|
| Capacity | ~4 chunks | Essentially unlimited |
| Duration | Seconds to minutes | Years |
| Access | Immediate | Requires retrieval |
| Fragility | Very fragile | Stable (with consolidation) |

Learning is the process of moving things from working memory into long-term memory. [[How We Learn]] explains this encoding process. [[Sleep and Memory Consolidation]] explains when consolidation actually happens.

## Why This Matters for Study

- Don't try to learn too many things in one session
- Prior knowledge dramatically increases effective working memory capacity
- [[The Feynman Technique]] forces you to compress knowledge into chunks, revealing gaps
- [[Active Recall]] works partly because successfully retrieving something *strengthens the chunk*
